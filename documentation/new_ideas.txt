New Feature Ideas
=================

1. VIDEO UPLOAD RISK HIGHLIGHTS
   - After video analysis completes, post-process the replay timeline to find
     peak injury risk moments (highest severity, longest duration)
   - Aggregate consecutive frames with same risk type into RiskSegments
   - Present top 5-10 risk moments as "jump to" markers on the video scrubber
   - Sort by severity (high first), then by duration
   - Already have RiskSegment type in types.ts and replay timeline caching

2. USER ACCOUNTS (SIMPLE FOR HACKATHON)
   - Local MongoDB for persistence
   - Simple registration: username only, no password (hackathon demo)
   - Store per-user: injury history, risk profile, past workout summaries
   - On login, load user's risk profile into the streaming pipeline
   - Schema: { username, injury_history[], risk_profile{}, workouts[] }

3. PERSONALIZED RISK THRESHOLDS
   - evaluate_injury_risks() in movement_quality.py currently uses hardcoded
     clinical thresholds (FPPA > 15 = medium, > 25 = high)
   - Make thresholds configurable per-user based on injury history
   - Example: ACL history -> lower valgus thresholds (10/18 instead of 15/25)
   - Example: Shoulder impingement -> add shoulder ROM risk checks
   - Pass user risk profile into MovementQualityTracker on session start
   - Risk profile structure:
     {
       "acl": { "side": "left", "severity": "tear", "years_ago": 2 },
       "shoulder": { "side": "right", "type": "impingement" },
       "lower_back": { "type": "herniated_disc", "level": "L4-L5" }
     }
   - Maps to threshold modifiers:
     acl_history -> FPPA thresholds * 0.65 (more sensitive)
     shoulder -> add angular_velocity threshold for shoulder joints
     lower_back -> add trunk_lean thresholds * 0.7

4. GEMINI CHAT AGENT FOR INJURY INTAKE
   - New chat endpoint: POST /api/chat with conversation history
   - Gemini 2.0 Flash takes user conversation as input
   - System prompt instructs Gemini to extract structured injury history
   - User says: "I tore my ACL a few years ago, my right shoulder clicks"
   - Gemini extracts: { acl: {side: "unknown", type: "tear", years_ago: 2},
     shoulder: {side: "right", type: "impingement_suspected"} }
   - Extracted profile saved to MongoDB under user account
   - Profile automatically adjusts risk thresholds for future sessions
   - Chat also provides personalized form cues based on injury history
   - Gemini can reference the active_guideline to give context-aware advice

5. WORKOUT HISTORY & TRENDS
   - After each session, save summary to user's account:
     { date, duration, movements[], risk_events[], fatigue_peak, form_scores }
   - Dashboard showing trends over time:
     - Is their ACL risk improving? (valgus angles decreasing)
     - Fatigue onset getting later? (endurance improving)
     - Form scores improving for specific movements?
   - Weekly/monthly progress reports generated by Gemini

6. RISK-AWARE FORM CUES
   - Combine the guideline system with personalized risk profile
   - Someone with ACL history doing squats gets extra emphasis:
     "Keep knees tracking over toes - IMPORTANT given your ACL history"
   - Form cues prioritized by user's vulnerability profile
   - Cues update in real-time as the system detects which movement they're doing

7. VOICE COACHING VIA ELEVENLABS TTS
   - Real-time spoken alerts when high injury risk is detected
   - ElevenLabs API for natural-sounding voice (not robotic TTS)
   - Trigger conditions: any injury_risk with severity "high", or sustained
     "medium" risk for >3 seconds
   - Example alerts:
     "Your knees are caving inward -- push them out over your toes"
     "Your trunk lean is getting excessive -- engage your core and slow down"
     "You're favoring your left side -- consider taking a break"
   - Cooldown between alerts (e.g. 8-10 seconds) so it doesn't spam the user
   - Personalized: reference the user's injury history in the alert
     "Watch your right ankle -- that landing pattern is stressing it"
   - Priority queue: highest severity risk spoken first, lower risks queued
   - Backend generates alert text from injury_risks + active_guideline + user
     profile, sends to ElevenLabs streaming API, plays audio in browser
   - Option to use Gemini to compose more natural/contextual alert phrasing
   - Mute toggle in the UI for when user doesn't want voice feedback

8. STREAMING PERFORMANCE IMPROVEMENTS
   - Video-skeleton overlay alignment has noticeable lag — the skeleton lags
     behind the video by the pipeline round-trip time (~50-100ms)
   - Currently using DelayedVideoCanvas to buffer video frames and display
     them delayed by EMA-smoothed RTT, but this introduces visual mushiness
   - Improvements needed:
     a) Tighter RTT estimation — track per-frame send/receive timestamps
        instead of EMA smoothing, reduce jitter in the delay buffer
     b) Reduce WebSocket overhead — batch multiple frames' results when
        backend is faster than frontend consume rate
     c) Frontend frame pacing — decouple canvas paint from WebSocket
        message arrival, interpolate skeleton between known keyframes
        to smooth over network jitter
     d) Backend pipeline parallelism — overlap YOLO inference with
        biomechanics computation (currently sequential per-frame)
     e) Measure and display actual overlay lag in DebugPanel so we can
        track improvement over time
   - Goal: skeleton overlay should feel "stuck to the body" at 30fps with
     no perceptible drift or rubber-banding

9. USER FLOW REDESIGN — TWO-PATH LANDING PAGE
   - Replace current flat demo video list with a guided two-path experience
   - Landing page shows TWO large cards side by side, full width:

     CARD 1: "Personal Workout Safety"
     - Image: single person doing a workout, skeleton overlay glowing
     - Default state: grayscale / desaturated, subtle breathing animation
     - On hover: transitions to full color, skeleton segments animate
       through a risk detection sequence (green -> yellow -> red on knee),
       card lifts with shadow, slight scale-up (1.02x)
     - Click leads to INDIVIDUAL FLOW

     CARD 2: "Team Performance Monitor"
     - Image: group of athletes on a field, multiple skeleton overlays
     - Same grayscale -> color hover treatment
     - On hover: fatigue bars animate filling up next to each player,
       one player's skeleton pulses red
     - Click leads to COACH FLOW

   INDIVIDUAL FLOW (Card 1):
   - Step 1: ONBOARDING — conversational injury intake
     - Chat interface with Gemini agent (see feature #4)
     - User describes past injuries naturally: "I fell while rock climbing
       and rolled my ankle forward, it still bothers me"
     - Gemini extracts structured profile, confirms back to user
     - "Skip" option for users who just want to explore without a profile
   - Step 2: CHOOSE YOUR MODE
     a) "Analyze a YouTube video" — paste URL, BRACE downloads and
        processes it against your injury profile
        - Results page: video player with risk-annotated timeline
        - Skeleton overlay on the video with segments that change color
          (green/yellow/red) as risk levels change
        - On-hover popovers on timeline risk markers: show a mini skeleton
          snapshot of that moment with the risky joint highlighted
        - Summary card: "This workout contains 3 high-risk segments for
          your ankle. Modifications suggested at 4:12, 7:30, 12:05."
     b) "Use my webcam" — live real-time analysis with personalized
        thresholds
     c) "Upload a video file" — same as YouTube but from local file
   - Step 3: DEMO VIDEOS (if user wants to explore without their own content)
     - Curated set: individual workout videos (squats, lunges, HIIT, yoga)
     - "Show more" filter reveals the full video library
     - Each demo video card has a mini skeleton preview that animates on
       hover, showing the movement type

   COACH FLOW (Card 2):
   - Step 1: Brief setup — team name, sport type (affects movement
     guidelines), number of expected players
   - Step 2: CHOOSE YOUR MODE
     a) "Use field camera" — live multi-person tracking with fatigue
        dashboard per player
     b) "Upload game footage" — post-game analysis with per-player
        fatigue curves and risk events
   - Step 3: DEMO VIDEOS (curated for multi-person)
     - Basketball drills, soccer practice, team training footage
     - "Show more" filter reveals the full video library (same pool, just
       different default filter)

   DESIGN DETAILS:
   - Cards use CSS filter: grayscale(100%) -> grayscale(0%) on hover
   - Transition: 0.4s ease-out for color, 0.3s for transform
   - Card images could be short looping video clips (3-5s) that play on
     hover instead of static images for extra wow factor
   - Mobile: cards stack vertically, tap to expand instead of hover
   - Both paths share the same analysis engine — the flow just pre-
     configures the UI layout (single-subject focused vs multi-subject
     dashboard) and filters the demo video library
